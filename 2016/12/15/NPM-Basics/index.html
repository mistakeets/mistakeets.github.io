NPM Basics

The NPM Basics tutorial on Treehouse was a very informative tutorial that provided many useful tips and commands related to working with npm packages.

How to compare packages that do similar things (such as bcrypt vs bcrypt nodes)

  1. Look at how often new versions are released (are there regular updates or is there no active development?)
  2. Look at how many contributors on GitHub
  3. Look at google ranking
  4. Look for testing information

package.json

—npm init
(The utility asks you all the questions related to generating a package.json file instead of manually editing the file)
—name the package, version number, test command, entry point, open source license etc

—save use to add an npm package to your dependency list on your package.json

.gitignore file so you can define things to ignore such as have your project ignore npm packets during a deployment/git push (no need to upload npm packages to your GitHub page, thats what the package.json is for)

—save-dev Dev dependency - define dependencies for developers / people planning to contribute to your project (packages not needed by end users)

Node_env = production npm install (install only reg dependency, will not install dev dependency)

Versioning

1.0.0
First digit = major release (1.0 to 2.0)
Second digit = minor release (such as 1.1)
Third digit = patch release (such as 1.0.1 or 1.1.3)

Use Tilda ~ in the package.json to only allow new patch releases to be installed (such if depend is ver 0.8.3, you could use 0.8.4 or 0.8.5, but not 0.9)

Use ^ to allow minor version (if you package.json has version 1.0 of something, but a new version is available, go ahead and allow npm update to install 1.1, but not 2.0)

Updating npm packages

npm update -g

Install / uninstall

npm install / npm uninstall

Install-all : install reg dep and dev dep

-g global install of the npm package

npm uninstall —save-dev (remove a dev dependency)

Npm uninstall -g (uninstall global)

Npm outdated - check to see if any packages need to be updated
