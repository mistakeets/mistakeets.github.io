Node.js

As part of the weekly project we also had a goal of learning more about Node. The Node.JS is a JavaScript runtime, which gives you the ability to run JavaScript outside of the web browser. This is very useful on the backend, as you can design network applications that run on a server.

Node.JS has its own runtime console, also known as a REPL (Read-Eval-Print-Loop). The behavior is somewhat similar to typing JavaScript into the Google Chrome console, or similar to working with Python where you have a runtime environment to execute your code.

Out of the box you have have various node specific functions and objects (the library / API) to work with. Each part of the API has various methods which provide additional functionality. For example, there is the File System module (fs). If you use the Readfile method included with fs, you can then read the contents of a file and load it into buffer. This could be useful for dealing with server log files, or going through a list of passwords (which could then be sent through a program to encrypt the passwords).

To learn more about Node, our team started with the NodeSchool tutorial “learnyounode”. Here is one example of code we worked with that reads a file directory, then checks for a particular file extension (basically “find me all the files of this kind in a directory”)

var fs = require('fs');
var path = require('path');

fs.readdir(process.argv[2], function (err, list) {
  list.forEach(function (filename) {
    if (path.extname(filename) === '.' + process.argv[3]) {
      console.log(filename);
    }
  });
});

In hindsight, Nodeschool is not the best starting point in my opinion. While the exercises are very good, they seem to assume a higher level of understanding / prior experience (such as familiarity with callbacks). Callbacks in particular can be difficult to wrap you head around, as now you are dealing with functions where the function will take another function as an argument.

Our team decided to take a step back and work through the Node.JS basics tutorial on Treehouse which we found to be a better starting point. The “ramp up” of the tutorial is slower, however it is still very dense with information. The tutorial works through various key concepts, and has an exercise that builds a program to parse through JSON objects.

Key Learning Concepts

  1. Node can perform tasks that are “non-blocking”, where multiple functions can be running at once. the functions are not “blocked” by other functions running. This is also known as asynchronous, where your program does not have to wait for step 1 of the program to finish before step 2 can start.

  2. Node is event driven, where you have requests and responses (an “emitter” and “listener”). This gives you flow control inside the program, where the listener function will call the callback function once a particular event has happened / once a condition has been met.
