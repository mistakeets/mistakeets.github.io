Express

Express was another topic reviewed during the project. Express is a web framework built using Node. One of the primary things that Express handles is setting up routes, which are endpoints used to trigger a set of programming. Express can be used where a client/server relationship is needed in your code.

As an example, you could create a route within your Express server process that could answer http requests (this would trigger an endpoint in your code to then send out articles to your website). Another example of a route is code that can send information to a database and update the contents of a database table.

The Express tutorial on Treehouse was a pretty good source of information, and provides a good starting point for learning about Express. Here is some example code from the tutorial which gives a sample of one way to setup a .js file when working with Express.

'use strict';

var express = require('express');
    posts = require('./mock/posts.json')

//set up jade template
app.set ('view engine', 'jade');
app.set ('views', _ _ dirname + './folder_path')

app.get('/', function(request,response){
    response.render('index')
});

app.get('/blog:title?', function(request, response){
    var title = request.params.title;
    if (title === undefined) {
        res.status(503);
        res.send("This page might be created someday.");
    }else {
        // grab post based on the route
        var post = posts[title] || {};
        response.render('posts', {post : post});
    }
});


}

app.listen(3000, function(){
    console.log("The frontend server is running on port 3000!")
});

Another interesting part about the express tutorial was that it also covered Jade templates. One could use Jade templates as a way to quickly create content for the web. Instead of typing out all the html, you can use block syntax (indent by whitespace) in Jade, and when using Jade in conjunction with Express, you can setup your Express server process to handle rendering all the Jade templates into html for usage on a website.
