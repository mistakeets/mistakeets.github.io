My team mate and I wanted to continue working on basic algorithms, so we agreed to meet up at the Learners Guild on Saturday. After attending a nice event on Friday night, I was ready to go back to the Guild on Saturday and tackle more challenges.

The first algorithm we worked on was a fun little challenge that I had seen before in previous programming courses: let’s find the longest word! This was very helpful to see in JavaScript as it gave me an opportunity to see how various syntax interacts when you build the function.

Find the Longest Word in a String code:

function findLongestWord(str) {
  var arr = str.split(" ");
  var count = arr[0];
  for (var i = 1; i <= arr.length-1; i++) {
    if(count.length < arr[i].length){
      count = arr[i];
    }
  }
  return count.length;
}

findLongestWord("The quick brown fox jumped over the lazy dog");

The next tutorial was a bit more challenging, and my team mate did an excellent job of helping me understand how to create a function that will capitalize the first letter of each word. At first we tried various ways to solve this problem, then my team mate figured out an effective solution using split, map, and slice.

Title Case a Sentence:

function titleCase(str) {

  return str.toLowerCase().split(' ').map(function(word) {
    return(word.charAt(0).toUpperCase() + word.slice(1));
  }).join(' ');
}

titleCase("I'm a little tea pot");

Pair programming was also useful for working on the next tutorial, which was to make a function that would return the largest numbers in arrays. By working together in a pair, we could look at the various ways the code could be written out, and which option was more effective. After bouncing a few ideas around, we ended up using a nested for loop within our function and would increment through the input to find the largest number in each array.

Return the Largest Numbers in Arrays:

function largestOfFour(arr) {
  var newArr = [];
  var tempArr = arr[0][0];
  for (var i = 0; i < arr.length; i++){
    tempArr = 0;
    for (var j = 0; j < arr[i].length; j++){
      if(tempArr < arr[i][j]){
        tempArr = arr[i][j];
      }
    }
    newArr.push(tempArr);
  }
  return newArr;
}

largestOfFour([[4, 5, 1, 3], [13, 27, 18, 26], [32, 35, 37, 39], [1000, 1001, 857, 1]]);

As a final tutorial for the day, we worked on an algorithm that checks if a string (first argument, str) ends with the given target string (second argument, target). The challenge behind this tutorial was to make function that would perform this check without using the method .endsWith()

This was another tutorial where my team mate really shined and showed his strength as a good problem solver. The solution he came up with was much more effective and elegant than my solution which had lots of loops(and at first my solution would not pass all the test cases).

Confirm the Ending:

function confirmEnding(str, target) {
  // "Never give up and good luck will find you."
  // -- Falcor

  return str.substr(str.length - (target.length)) === target;
}

confirmEnding("Bastian", "n");

Arrays and Basic Algorithms

After working on a few more tutorials, today my team mate and I got to move on to something a bit more exciting: Basic Algorithms. The first two tutorials (reverse a string, and factorial numbers) were pretty familiar to me from previous courses I have taken, and my team mate was also comfortable with these exercises so we were feeling pretty good. The next algorithm we worked on was a more daunting task: creating a function that would identify palindromes (words or phrases that are the same when spelled forwards or backwards, such as race car).

One of the real struggles was using a regular expression (regex) to make the input all lowercase and remove the non alpha numeric characters. We had a pretty good solution setup however it would never pass the test cases because our regex expression was wrong. Once we figured out the regex issue the code would pass all test cases.

Palindrome code:

function palindrome(str) {
// make all letters lowercase and remove non-alphanumeric characters
  str = str.toLowerCase().replace(/[^a-z|0-9]/g, "");

  // if the length of the string is 0 then it is a palindrome
  if (str.length === 0){
    return true;
  }
  // if the first letter and the last letter of the string
  // do not equal each other, then it is not a palindrome
  if (str[0] !== str[str.length-1]){
    return false;
  }
  //Else, run the function without the first and last characters.
  else{
    return palindrome(str.slice(1,str.length - 1));
  }
}

After reading through a few web sites about blog frameworks, Hexo seemed to be a good choice as it is JavaScript based, and required the installation of Node.JS in order to install and work properly. Hey great! We can install Node so that when we start learning Node it will already be installed. What a pain though; first there were issues after installing the Node Version Manager (nvm): command not found.

Ok so now nvm is installed…but I can’t run it? What the heck? After a few web searches I was able to find information about editing the terminal file (.bash_profile) to point at nvm. Cool nvm works now! Installed Node, and used the Node Package Manager (npm) to install Hexo. This is about where I ran out of steam and went home. Javascript Basics

Today I worked on basic JS tutorials with my team mate to review syntax and fundamentals before moving into more complex tutorials. It was wonderful working on tutorials with a team mate in a pair programming environment as there was a support system if either one of us had questions, and we both kept each other motivated to keep learning.

One thing I learned today was compound assignments (augmented math) work in JavaScript! Now if I want to I can type:

x += y

instead of…

x = x+y

Another handy little thing I learned was typing \” allows you to put double quotes inside a string. I actually liked this syntax better than what I had previously learned in Python. Within Python I used to type triple quotes (then double quotes inside the triple quotes) to achieve the same result. Out of curiosity I did open a Python environment to see if \” works in Python and WHOA IT DOES! Sometimes even the little things can be exciting.

After working through tutorials to review concepts such as how to find the length of a string, or how to use bracket notation to find a character within a string, some more very important concepts started to be presented: my team mate and I started working with arrays.

Out of all the tutorials I worked on today with my team mate, I felt the most valuable were the tutorials that taught array.push(), array.pop(), array.shift(), and array.unshift().

How to append data to the end of an array using push:

var myArray = [["John", 23], ["cat", 2]];
myArray.push(["dog", 3]);

How to remove the last item from an array using pop:

var myArray = [["John", 23], ["cat", 2]];
myArray.pop();

How to remove the first item from an array using shift:

var myArray = [["John", 23], ["dog", 3]];
myArray.shift();

How to add an element to the beginning of an array using unshift:

var myArray = [["John", 23], ["dog", 3]];
myArray.unshift(["Paul", 35]);

In the afternoon my team mate and I worked on tutorials to review functions. It was good to read up on global scope vs. local scope, and there was a fun challenge where we built a function that would do Blackjack card counting. The card counting function was an “a ha” moment for me as I remembered a neat little trick I had previously learned: the conditional (tenary) operator.
